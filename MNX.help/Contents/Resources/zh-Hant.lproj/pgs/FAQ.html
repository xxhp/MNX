MNX 輔助說明

常見問題

MNX 可以做什麼？

Mainnav 目前並沒有推出 Mac OS X 平台的相關軟體，而 MNX 的用途，就在於幫您讀取 Mainnav 的 GPS 裝置上的資料，瀏覽資料內容，並且轉存成各種通行的檔案格式，包括 GPX、KML 與 TCX。您可以在 Mainnav 網站上，了解更多 Mainnav 的 GPS 產品的相關訊息。

MNX 不能做什麼？

MNX 最主要的用途就是從 GPS 裝置讀取與轉存資料，因此，許多軟體有體重、BMI 等體能資料追蹤功能、幫你計算消耗了多少卡路里…等，MNX 沒養這些功能，也不會幫你自動同步資料。而由於 MNX 在儲存與載入資料方面效能也不是很好，這部份我沒有花多少力氣寫，所以也建議不要在 MNX 儲存太多資料。

由於主要功能就是讀取與轉存資料，因此建議您在轉出資料後，使用其他軟體管理資料，或是上傳到Garmin Connect 、 EveryTrail 、 RunKeeper 、 Endomondo 等網站。基本上，讓雲端儲存、管理您的運動資料，會比單機作業好，而且您還可以再透過 Facebook 分享運動或旅遊記錄

為什麼沒有提供像是 POI 列表這些功能呢？

MNX 基本上是參靠 mainnav-reader 計畫所做的反向工程，解讀出 Mainnav 裝置上的資料格式寫出來的。Mainnav reader 透過 python 語言讀取資料，而由於我自己想要有一套有圖形介面的軟體，所以才寫了 MNX。

由於是透過逆向工程解讀資料，Mainav 本身也沒有公開文件，自然也只能做出目前已經解讀出來的部分功能。要能夠使用全部的功能，還是要等 Mainnav 推出 Mac OS X 版本的軟體。

MNX 支援哪些裝置？

我自己只有一隻 MG-600，而且只用 USB 介面傳輸。如果您的裝置可以用，或是不能用，可以透過 GitHub 的 issue-tracker 系統與大家分享。
